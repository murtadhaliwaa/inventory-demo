# أمن المشروع وقائمة ما قبل النشر

> هذا المستند **إرشادي**؛ لا يغني عن مراجعة أمنية مناسبة لبيئة الإنتاج.

## 1) أسرار وبيئة

| القاعدة | التفاصيل |
|---------|-----------|
| لا ترفع `.env` | مذكور في `.gitignore` — راجع `git status` قبل كل commit |
| لا توثّق أسرارًا في `تقارير/` | اذكر أسماء المتغيرات فقط |
| `DATABASE_URL` | يصل إليه خادم Next عبر Prisma؛ لا يُعرض في الواجهة |
| مفاتيح `NEXT_PUBLIC_*` | ظاهرة للمتصفح — **لا** تضع فيها أسرار خادم |

## 2) مصادقة المستخدم

- **Edge:** `middleware.ts` يتحقق من الجلسة عبر Supabase.  
- **الخادم:** `requireUser()` قبل أي عملية حساسة في `inventory.ts`.  
- **وضع التطوير:** تجاوز وهمي عند غياب Supabase — **عطّل الاعتماد عليه في الإنتاج** (تأكد من ضبط `.env`).

## 3) صلاحيات الحذف والأدوار

- **`src/lib/auth/roles.ts`**: أدوار `admin` | `operator` | `viewer` من `app_metadata.wms_role` أو `wms_admin`.
- **`WMS_ADMIN_EMAILS`**: عند ضبطه يُقيَّد **الحذف** بهذه البريدات فقط.
- **`requireManageUser` / `requireDeleteUser`**: في كل Server Action حساسة.
- **سجل التدقيق**: `/audit` للمشرفين — جدول `audit_logs`.

## 4) قاعدة البيانات وPrisma

- الاتصال عبر **pooler** (Supabase) مناسب للتطبيق؛ الترحيلات عبر **`DIRECT_URL`** (انظر `prisma.config.ts`).  
- **RLS في Postgres:** إن كان الاتصال بصلاحيات تتجاوز RLS، فسياسات Supabase لا تحمي قراءة Prisma — وثّق ذلك في تصميمك أو استخدم دورًا محدودًا إن لزم.

## 5) المدخلات والهجمات الشائعة

- كل مدخل من الواجهة: **Zod** + حدود أرقام معقولة (موجودة في مخططات الكميات).  
- تجنب `dangerouslySetInnerHTML` لمحتوى المستخدم.  
- عند إضافة رفع ملفات: حجم، نوع، تخزين آمن، وفحص فايروس (خارج نطاق هذا المشروع حاليًا).

## 6) التبعيات (npm)

- راجع `npm audit` دوريًا عند التحديثات الكبرى.  
- ثبّت الإصدارات في `package-lock.json` ولا تحذف القفل عشوائيًا.

## 7) قائمة قبل النشر (Production)

- [ ] `.env` في الاستضافة يحتوي `NEXT_PUBLIC_SUPABASE_*` + `DATABASE_URL` + `DIRECT_URL`  
- [ ] `WMS_ADMIN_EMAILS` مضبوط إن أردت تقييد الحذف  
- [ ] `npx prisma migrate deploy` مطبّق على قاعدة الإنتاج  
- [ ] `npm run build` ناجح  
- [ ] جلسة تسجيل دخول حقيقية تُختبر على النطاق النهائي  
- [ ] لا تسرّبات في استجابات الأخطاء للمستخدم النهائي (رسائل عامة في الإنتاج إن لزم)
